# Code of Conduct

## Our Standard

This project is intended to be a professional, research-friendly, and respectful space.

Examples of expected behavior:

- give technical feedback directly and respectfully
- assume good faith while still being precise about risks and flaws
- focus discussion on code, evidence, reproducibility, and user impact
- keep disagreements substantive and non-personal

Examples of unacceptable behavior:

- harassment, threats, or personal attacks
- discriminatory or demeaning language
- bad-faith disruption, intimidation, or dogpiling
- publishing private information without permission

## Scope

This Code of Conduct applies to repository discussions, issues, pull requests, and other project-managed collaboration spaces.

## Enforcement

Project maintainers may remove comments, close discussions, or restrict participation when behavior makes the project harder to collaborate on safely and productively.

## Reporting

For conduct concerns, open a private security-style report if the platform supports it, or contact the maintainer directly before escalating in public.
